口径说明：
- 稳态计时：脚本先用牺牲 scope 预热一发（建连/装 Lua 脚本/故障检测全部发生在计时窗外），
  精度只量稳态；冷启动与检测成本单独上报为"首发请求耗时"。
  旧凭证 m2_ratelimit_retest.txt 的 0.76%（521/525）为含冷启动的旧口径，保留不改。
- 故障检测延迟实测 1046ms，上界 = socket_connect_timeout 1s + 事件循环开销；
  检测窗内请求延迟但不失败（fail-open 降级），检测后全员瞬时切换本地桶（粘滞降级，
  每 5s 一个顺路探针试探恢复）。
- 停 Redis 形态 = docker stop 容器（Windows + Docker Desktop 下表现为 SYN 静默丢包，
  走满 connect 超时；若为端口拒绝形态则检测为毫秒级）。
- 计数口径 = 按 HTTP 调用（§10.1 #11 冻结项）；参数 rate=50/s、capacity=25（开局满桶）、
  20 协程 × 10s，理论放行 = 25 + 50×10 = 525；单副本，降级本地配额 = 全局/1。
- 环境：Windows 11 本地 + Docker Desktop Redis 7（127.0.0.1:6379/db9）；
  在线模式时钟源 = Redis TIME（服务端原子），降级模式 = time.monotonic()。
- 脚本 = 本提交版 scripts/loadtest_ratelimit.py（预热 + 模式/首发耗时上报）。
